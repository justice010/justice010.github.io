Vulnhub打靶之DC-1靶场

1. 扫描C段，发现主机

执行命令 ifconfig，查询kali的IP地址

使用 nmap 扫描C段，发现目标主机。

2. 扫描靶机开放端口

使用 nmap 扫描目标主机，发现80端口开放。

3. 访问网站

打开浏览器访问目标网站，添加 Wappalyzer 插件，识别出该网站使用 Drupal 7。


4. 利用 Metasploit 搜索 Drupal 可利用模块

进入 Kali 中的 Metasploit，搜索 Drupal 相关的可利用模块。


我这里用的是第二个模块

5. 利用漏洞成功执行 shell

成功利用漏洞，执行命令 python -c 'import pty;pty.spawn("/bin/bash")' 进入交互式 shell。

6. 列出文件，找到 flag1

在交互式 shell 中执行 ls 命令，找到 flag1。

7. 寻找 Drupal 配置文件

根据 flag1 的提示，寻找网站 CMS 的配置文件。Drupal 的配置文件通常位于 /sites/default/settings.php，找到 flag2。

8. 利用数据库提供的用户名和密码

使用数据库用户名和密码进入数据库，发现 Drupal 数据库和 users 表。在 users 表中找到 admin 用户和密码，但密码已被加密。



9. 加密明文密码

在 /scripts 文件夹中发现加密脚本 “password-hash.sh”，对明文 “123456” 进行加密，注意需要复制缺失的 “includes” 文件夹。

10. 更新管理员密码

进入数据库，执行命令 update users set pass='密文' where name='admin'; 修改管理员用户的密码。

11. 登录网站

使用新的管理员账号密码登录网站，获得 flag3。

12. 利用 find 进行 SUID 提权

根据 flag3 提示，使用 find 进行 SUID 提权，成功获得 root 权限。

13. 获得最后一个 flag

获得 root 权限后，找到最后一个 flag。
